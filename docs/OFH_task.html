daily_bookings_graph <-  function(data) {
  data |> 
    ggplot(aes(x = day, y = daily_bookings, colour = clinic)) +
    geom_line() +
    geom_ribbon(aes(ymin = lower_daily_bookings, ymax = upper_daily_bookings, fill = clinic), line = NA, alpha = 0.12) +
    labs(
      title = "Daily bookings",
      x = "days since invitation",
      y = "bookings"
    )
}

cumul_bookings_graph <- function(data){

}

DT_weibull_k      <- 0.7 # k < 1 so decreasing over time (rapid early bookings, slowing tail)
DT_weibull_lambda <- 2.5 # Higher lambda means bookings accumulate more slowly overall.

NPL_weibull_k      <- 0.5
NPL_weibull_lambda <- 3

days <- 0:63  # 5 weeks pre-clinic
w1_sends <- tibble(
  clinic    = c("clinic 1", "clinic 2"),
  mailing_type = c("NPL", "DT"),
  send_day  = c(8, 3),
  mailing_size = c(10000, 5000),
  conversion_rate = c(C1_mailings |> filter(type == "NPL") |> pull(convMid), 
                      C2_mailings |> filter(type == "DT") |> pull(convMid)),
  lower_conversion_rate = c(C1_mailings |> filter(type == "NPL") |> pull(convLow), 
                      C2_mailings |> filter(type == "DT") |> pull(convLow)),
  upper_conversion_rate = c(C1_mailings |> filter(type == "NPL") |> pull(convHigh), 
                      C2_mailings |> filter(type == "DT") |> pull(convHigh))
)

wave1_booking_curve <- w1_sends |>
  expand_grid(day = 0:63) |>
  mutate(
    days_since_send = day - send_day,
    proportion = case_when(mailing_type == "NPL" 
                                ~ pweibull(days_since_send, shape = NPL_weibull_k, scale = NPL_weibull_lambda),
                                mailing_type == "DT"
                                ~ pweibull(days_since_send, shape = DT_weibull_k, scale = DT_weibull_lambda)),
    cumul_bookings = proportion * mailing_size * conversion_rate,
    lower_cumul_bookings = proportion * mailing_size * lower_conversion_rate,
    upper_cumul_bookings = proportion * mailing_size * upper_conversion_rate
  ) |>
  mutate(
    daily_bookings = case_when(
      days_since_send == 0 ~ 0,
      days_since_send == 1 ~ cumul_bookings,
      .default = cumul_bookings - lag(cumul_bookings, n = 1)
    ),
    lower_daily_bookings = case_when(
      days_since_send == 0 ~ 0,
      days_since_send == 1 ~ lower_cumul_bookings,
      .default = lower_cumul_bookings - lag(lower_cumul_bookings, n = 1)
    ),
    upper_daily_bookings = case_when(
      days_since_send == 0 ~ 0,
      days_since_send == 1 ~ upper_cumul_bookings,
      .default = upper_cumul_bookings - lag(upper_cumul_bookings, n = 1)
    ),
    .by = clinic
  )

General approach

5 waves of mailings, with the fifth wave taking place during the first week of clinics opening.

NPL vs DT

I’ve assumed the two clinics serve distinct but similar populations. I assumed that each clinic did not have overlap in the mailout reach or sign ups from each clinic.

As a result, I decided to do first mailing for clinic 2 with 5000 DT mailings which would give information on conversion rate and time-to-booking to decide waves for clinic 1 for the rest of the scenario.

After that, I looked to balance the number of sign ups for each clinic based on the initial conversion rates provided.

I checked if the clinics could be filled with the assumed conversion rate, and then balanced the number of NPL mailings across the two clinics so both could reach a full capacity.

This led to clinic 1 having 40,000 mailings for NPL and DT, while clinic 2 had 90,000 NPL mailings and 5,000 DT mailings. Based in the assumed conversion rate midpoint, clinic 1 would have 2200±600 people, while clinic 2 would have 2187.5±715.

Number of bookings and conversion rate

I’ve projected the number of bookings per wave based on the provided conversion rates. In practice I would update this using a Bayesian approach specific to each clinic and mailing route, updating in response to new information from each wave.

Time-to-booking

I’ve made assumptions around the time it takes people to respond to mailings and book their appointment, using Weibull distribution curves to determine daily booking volumes. The shape and steepness was chosen so that DT would give a quicker response (under the assumption that communications from GP surgeries carry more weight). The actual shape and slope of the curve is guess work on my part, I assumed most people would book within 5 days for DT and 8 for NPL.

Timing of waves

I’ve used the plateau point of 80% of predicted bookings on the time-to-book curve to predict the send out date of each successive wave. The first DT wave of mailings for clinic 1 was placed in wave 3 to help ensure enough information would have come through from the test wave of DT at clinic 2.

Readjusting the plan

I don’t think this current plan would work that well without readjustment. Especially since the width of the estimates are relatively wide.

I’d track conversion rates for each clinic and mailing method using a Bayesian approach, weighting recent data more heavily if rates appear to be shifting. These updated estimates feed into a predicted plateau for each wave. By fitting a new Weibull curve and reading off the asymptote, we can anticipate the final booking count and shift subsequent delivery dates earlier or later accordingly. We may need to define the plateau conservatively given the lag between booking and send, but having 5 waves should give us enough runway to absorb that uncertainty.

Other assumptions

  • Targetting of particular demographics has been ignored. If it was, I’d use a random effects model taking in a range of demographic factors, probably from census data.

  • The effect of mailing out too early on conversion and time-to-booking has been ignored. In practice I’d try to glean this from past data, maybe with an initial regression model with a number of different factors. Then swap that out for a Bayesian approach once mailings have begun.

  • Budgetary constraints are not a main consideration. I assumed that the cost of mailouts was less that the cost of an empty booking spot. If we needed to balance this I’d look at a newspaper vendor model to decide how to balance costs of empty vs filled spots and how much spend is necessary to fill a spot.

  • No repeat mailings. I haven’t included the effect of repeat mailings without any data on the potential increase in conversion rates, and since we can’t track address level responses for NPL.

  • No sms/email follow ups. This was because of no initial data, in reality I’d track conversion rates and booking capacity, then implement sms/email follow if the clinics are not filling from the letters sent pre-opening. If we could see the number of people consenting but not booking, it may be easier to build into the plan earlier and reduce the size of later mailing waves.

Invitation strategy for a new clinic

Non-Personalised Letters (NPL)

Sent to households based on address data (Royal Mail PAF). These can be sent multiple times to the same address but we cannot track responses at address level.

  • Minimum batch size: 4,000 addresses

  • Lead time: ~8 days from request to delivery

  • Can be sent multiple times to the same address

  • Cost: 50p per letter

  • ~1.5-3.0% of households book an appointment

  • Limited envelope stock: 130,000 available

DigiTrials (DT)

Personalised letters sent via the NHS to eligible individuals. These are sent once per person and the NHS applies eligibility filters before sending.

  • NHS filters out ineligible individuals (actual sends may be lower than requested)

  • Lead time: ~3 days from request to delivery

  • Only sent once to the same address

  • Cost £1.50 per letter

  • ~2.5-4.0% of individuals invited book an appointment

  • Available eligible population in target area:

    • ~40,000 individuals for clinic 1

    • ~5,000 individuals for clinic 2

Week 1–4: Each clinic has capacity for 500 appointments per week (i.e. 1,000 total per week across both clinics)

Your goal is to ensure the clinics are well-filled across their first 4 weeks

Please prepare a short plan outlining:

  • How you would approach invitation planning for these clinics

  • How you would allocate between NPLs and DigiTrials

  • Rough estimates of invitation volumes and timing

  • Key assumptions you are making

  • How you would monitor performance and adjust your plan

#| code-summary: wave 1 code
DT_weibull_k      <- 0.7 # k < 1 so decreasing over time (rapid early bookings, slowing tail)
DT_weibull_lambda <- 2.5 # Higher lambda means bookings accumulate more slowly overall.

NPL_weibull_k      <- 0.5
NPL_weibull_lambda <- 3

days <- 0:63  # 5 weeks pre-clinic
w1_sends <- tibble(
  clinic    = c("clinic 1", "clinic 2"),
  mailing_type = c("NPL", "DT"),
  send_day  = c(8, 3),
  mailing_size = c(10000, 5000),
  conversion_rate = c(C1_mailings |> filter(type == "NPL") |> pull(convMid), 
                      C2_mailings |> filter(type == "DT") |> pull(convMid)),
  lower_conversion_rate = c(C1_mailings |> filter(type == "NPL") |> pull(convLow), 
                      C2_mailings |> filter(type == "DT") |> pull(convLow)),
  upper_conversion_rate = c(C1_mailings |> filter(type == "NPL") |> pull(convHigh), 
                      C2_mailings |> filter(type == "DT") |> pull(convHigh))
)

wave1_booking_curve <- w1_sends |>
  expand_grid(day = 0:63) |>
  mutate(
    days_since_send = day - send_day,
    proportion = case_when(mailing_type == "NPL" 
                                ~ pweibull(days_since_send, shape = NPL_weibull_k, scale = NPL_weibull_lambda),
                                mailing_type == "DT"
                                ~ pweibull(days_since_send, shape = DT_weibull_k, scale = DT_weibull_lambda)),
    cumul_bookings = proportion * mailing_size * conversion_rate,
    lower_cumul_bookings = proportion * mailing_size * lower_conversion_rate,
    upper_cumul_bookings = proportion * mailing_size * upper_conversion_rate
  ) |>
  mutate(
    daily_bookings = case_when(
      days_since_send == 0 ~ 0,
      days_since_send == 1 ~ cumul_bookings,
      .default = cumul_bookings - lag(cumul_bookings, n = 1)
    ),
    lower_daily_bookings = case_when(
      days_since_send == 0 ~ 0,
      days_since_send == 1 ~ lower_cumul_bookings,
      .default = lower_cumul_bookings - lag(lower_cumul_bookings, n = 1)
    ),
    upper_daily_bookings = case_when(
      days_since_send == 0 ~ 0,
      days_since_send == 1 ~ upper_cumul_bookings,
      .default = upper_cumul_bookings - lag(upper_cumul_bookings, n = 1)
    ),
    .by = clinic
  )

### NPL vs DT

I've assumed the two clinics serve distinct but similar populations. I assumed that each clinic did not have overlap in the mailout reach or sign ups from each clinic.

As a result, I decided to do first mailing for clinic 2 with 5000 DT mailings which would give information on conversion rate and time-to-booking to decide waves for clinic 1 for the rest of the scenario.

After that, I looked to balance the number of sign ups for each clinic based on the initial conversion rates provided.

I checked if the clinics could be filled with the assumed conversion rate, and then balanced the number of NPL mailings across the two clinics so both could reach a full capacity.

This led to clinic 1 having 40,000 mailings for NPL and DT, while clinic 2 had 90,000 NPL mailings and 5,000 DT mailings. Based in the assumed conversion rate midpoint, **clinic 1** would have **2200±600 people**, while **clinic 2** would have **2187.5±715**.

### Number of bookings and conversion rate

I've projected the number of bookings per wave based on the provided conversion rates. In practice I would update this using a Bayesian approach specific to each clinic and mailing route, updating in response to new information from each wave.

### Time-to-booking

I've made assumptions around the time it takes people to respond to mailings and book their appointment, using Weibull distribution curves to determine daily booking volumes. The shape and steepness was chosen so that DT would give a quicker response (under the assumption that communications from GP surgeries carry more weight). The actual shape and slope of the curve is guess work on my part, I assumed most people would book within 5 days for DT and 8 for NPL.

### Timing of waves

I've used the plateau point of 80% of predicted bookings on the time-to-book curve to predict the send out date of each successive wave. The first DT wave of mailings for clinic 1 was placed in wave 3 to help ensure enough information would have come through from the test wave of DT at clinic 2.

### Readjusting the plan

I don't think this current plan would work that well without readjustment. Especially since the width of the estimates are relatively wide.

I'd track conversion rates for each clinic and mailing method using a Bayesian approach, weighting recent data more heavily if rates appear to be shifting. These updated estimates feed into a predicted plateau for each wave. By fitting a new Weibull curve and reading off the asymptote, we can anticipate the final booking count and shift subsequent delivery dates earlier or later accordingly. We may need to define the plateau conservatively given the lag between booking and send, but having 5 waves should give us enough runway to absorb that uncertainty.

### Other assumptions

-   **Targetting of particular demographics** has been ignored. If it was, I'd use a random effects model taking in a range of demographic factors, probably from census data.

-   The effect of **mailing out too early** on conversion and time-to-booking has been ignored. In practice I'd try to glean this from past data, maybe with an initial regression model with a number of different factors. Then swap that out for a Bayesian approach once mailings have begun.

-   **Budgetary constraints** are not a main consideration. I assumed that the cost of mailouts was less that the cost of an empty booking spot. If we needed to balance this I'd look at a newspaper vendor model to decide how to balance costs of empty vs filled spots and how much spend is necessary to fill a spot.

-   **No repeat mailings**. I haven't included the effect of repeat mailings without any data on the potential increase in conversion rates, and since we can't track address level responses for NPL.
